## 1. Nice-to-have (после MVP)

Пул, из которого берётся следующая задача по желанию — не роадмап с датами.

| Фича | Что даёт | Учебная нагрузка |
|---|---|---|
| **Firefox-порт** | Расширение работает и в Firefox | Различия MV3 Firefox vs Chrome (`browser.*` namespace, `webextension-polyfill`, упаковка `web-ext`) |
| **Кастомные промпты/шаблоны сводки** | «TL;DR», «по пунктам», «для эксперта», свой промпт | Хранение шаблонов в `chrome.storage`, подстановка в запрос, UI выбора |
| **История сводок** | Просмотр прошлых сводок | Локальное хранение в `chrome.storage`/IndexedDB, список, поиск по заголовку |
| **Экспорт в заметки** | Сводка → Markdown/буфер обмена/файл | `navigator.clipboard`, Blob-download, формат Markdown |
| **Стриминг ответа LLM** | Токен-за-токеном в UI, отзывчивость | Разбор SSE-стрима (`data:` … `[DONE]`) через `fetch` + `ReadableStream` |
| **Выбор языка сводки** | Сводка на нужном языке независимо от языка страницы | Параметр в промпте + настройка |
| **Локальный фейк-endpoint для интеграционных тестов** | Детерминированный прогон стека без реального ключа | Мини OpenAI-совместимый мок (напр. в контейнере) — вернуться к вопросу о Docker точечно |

## 2. Отложенные риски

- **Изменения политики Manifest V3 / Chrome Web Store.** Правила по content
  scripts, `host_permissions`, ревью store могут меняться. Митигирование:
  минимальные разрешения (сейчас `activeTab` + `storage`), без лишних хостов.
- **CORS/сетевые ограничения к пользовательскому endpoint.** У части провайдеров
  могут быть нюансы; при появлении — точечно добавлять `host_permissions`.
- **Лимиты токенов и стоимость у пользователя.** Длинные страницы → усечение
  (Этап 3); в post-MVP — чанкинг/выбор модели.
- **Приватность.** В LLM уходит текст, который пользователь сам отправил на
  саммаризацию. Явно коммуницировать это в README/UI; ничего не отправлять
  фоново без действия пользователя.

## 3. Excluded — Never (сознательно вне скоупа навсегда)

Эти пункты противоречат сути проекта («узкий клиентский саммаризатор, $0 инфра,
не Shelf») и не рассматриваются:

- **Библиотека / граф знаний / граф заметок.** Это **Shelf** — отдельный крупный
  проект (`~/Projects/ai-projects/shelf`). Не строить здесь библиотеку книг,
  полки, связи между выделениями. `aiBrowserPlagin` остаётся узким
  саммаризатором.
- **Локальная БД, эмбеддинги, векторный/гибридный поиск, RAG.** Вся «умная»
  индексация — тоже территория Shelf. Тащить SQLite/`sqlite-vec`/RAG в
  клиентское расширение-саммаризатор — неоправданное усложнение.
- **Собственный бэкенд-сервер / прокси ключей / хостинг.** Ломает принцип
  «$0 инфра, BYOK, никакого сервера». Ключ пользователя не проходит через
  чужой сервер — только прямой вызов из браузера.
- **Телеметрия / аналитика / «звонки домой».** Приватность и доверие — принцип.
- **Аккаунты / подписки / SaaS-биллинг.** Проект не бизнес; монетизация не цель.
