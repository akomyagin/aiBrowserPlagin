## 1. Что это

`aiBrowserPlagin` — браузерное расширение (Manifest V3, Chrome/Chromium),
**AI-саммаризатор веб-страниц и PDF**. Пользователь одним кликом (или выделив
фрагмент) получает сжатую сводку текущей страницы: расширение извлекает текст,
прогоняет его через LLM-API **по ключу самого пользователя (BYOK)** и показывает
результат во всплывающем окне (popup), а позже — в боковой панели (side panel).

Ключевое: **бэкенда нет вообще.** Вся обработка — прямой вызов
OpenAI-совместимого API из service worker расширения. Ни сервера, ни БД, ни
хостинга ключей — целевая инфраструктура автора ≈ **$0/мес**.

## 2. Зачем (и почему именно так)

Это **соло pet-проект соло-разработчика-интроверта** с приоритетом **тренировки
кодирования** (TypeScript, экосистема Manifest V3, Vite-сборка расширений).
Экономическая целесообразность **не обязательна** — это учебный/личный
инструмент, а не бизнес.

Принципы:

- **Нулевая инфраструктура** — расширение полностью клиентское, расходы автора
  ≈ $0/мес.
- **BYOK** — пользователь приносит свой ключ к OpenAI-совместимому endpoint.
  Ключ живёт только в `chrome.storage` браузера, **никогда** не хардкодится, не
  попадает в git и не логируется.
- **Узкий скоуп** — это простой быстрый саммаризатор, а не платформа. Сознательно
  не раздуваем (см. §6 и `POST_MVP_PLAN.md`).
- **Минимум общения** — самообслуживание, без саппорт-чата.

## 3. Чем это НЕ является (отличие от Shelf)

В портфеле есть родственный, но **отдельный** и куда более крупный проект
**Shelf** (`~/Projects/ai-projects/shelf`) — локальная «умная библиотека» книг и
заметок (Tauri-десктоп, SQLite + `sqlite-vec`, RAG с цитатами, граф заметок).

`aiBrowserPlagin` — это **НЕ Shelf и не его модуль.** Здесь:

- **нет** библиотеки книг, полок, импорта EPUB;
- **нет** графа заметок и связей между выделениями;
- **нет** локальной БД, эмбеддингов, векторного/гибридного поиска, RAG.

Это максимально простой инструмент «выделил → получил сводку». Не тащить сюда
архитектуру Shelf — это неоправданное усложнение (зафиксировано в
`POST_MVP_PLAN.md`, раздел «Excluded — Never»).

## 5. Этапы (обзор)

Полная разбивка с deliverable и критериями приёмки — в
[`TECHNICAL_PLAN.md §8`](./TECHNICAL_PLAN.md). Здесь — однострочный обзор.

**Фаза 1 (ядро MVP):**

- **Этап 0 — Bootstrap.** Каркас расширения: `package.json`, Vite + `@crxjs`,
  `manifest.config.ts` (MV3), заглушки `background`/`content`/`popup`. Сборка
  зелёная, расширение загружается в браузер. **(готово)**
- **Этап 1 — Саммаризация всей страницы.** Извлечение читаемого текста, вызов
  LLM из service worker, показ сводки в popup. Message-passing content→bg→popup.
- **Этап 2 — Настройки (BYOK).** Форма настроек в popup: base URL, ключ, модель;
  хранение через `chrome.storage.local`; дружелюбные ошибки при отсутствии ключа.
- **Этап 3 — Устойчивость и UX.** Обработка ошибок сети/лимитов, усечение
  слишком длинного текста под лимит токенов, индикатор загрузки, отмена.

**Фаза 2 (продолжение MVP):**

- **Этап 4 — Поддержка PDF.** Извлечение текста из PDF (`pdf.js`), в т.ч. из
  встроенного вьюера браузера.
- **Этап 5 — Саммаризация выделенного.** Контекстное меню (правая кнопка) →
  сводка только выделенного фрагмента.
- **Этап 6 — Side panel.** Chrome Side Panel API как альтернатива popup для
  комфортного чтения длинных сводок. **Конец MVP.**
