# CLAUDE.md

Инструкции для AI-сессий (Claude Code) в репозитории `aiBrowserPlagin`. Эти
инструкции имеют приоритет над поведением по умолчанию — следуй им точно.

## Что это за репозиторий

`aiBrowserPlagin` — браузерное расширение (**Manifest V3**, TypeScript):
**AI-саммаризатор веб-страниц и PDF**. Извлекает текст страницы/выделения →
прогоняет через LLM-API **по ключу пользователя (BYOK)** → показывает сводку в
popup (позже — в side panel). **Бэкенда нет** — прямой `fetch` из service
worker. Соло pet-проект соло-разработчика с приоритетом **тренировки
кодирования**; экономическая целесообразность не обязательна; инфраструктура
автора ≈ $0/мес.

**Это НЕ Shelf.** Shelf (`~/Projects/ai-projects/shelf`) — отдельный крупный
проект (Tauri, SQLite, RAG, граф заметок). Здесь — узкий саммаризатор **без**
библиотеки книг, графа, локальной БД, эмбеддингов и RAG. Не тащить сюда
архитектуру Shelf.

Видение — [`docs/PLAN.md`](docs/PLAN.md); техплан и этапы —
[`docs/TECHNICAL_PLAN.md`](docs/TECHNICAL_PLAN.md); за пределами MVP —
[`docs/POST_MVP_PLAN.md`](docs/POST_MVP_PLAN.md).

## Структура репозитория

| Путь | Содержимое |
|---|---|
| `docs/` | `PLAN.md`, `TECHNICAL_PLAN.md`, `POST_MVP_PLAN.md` |
| `.claude/skills/browser-extension-dev/SKILL.md` | Конвенции именно этого проекта для написания кода |
| `manifest.config.ts` | Типизированный Manifest V3 (`defineManifest`) — источник версии/путей |
| `vite.config.ts` | Vite + `@crxjs/vite-plugin` |
| `src/background/` | Service worker: приём `SUMMARIZE`, вызов LLM |
| `src/content/` | Content script: извлечение текста/выделения, PDF |
| `src/popup/` | Popup UI (голый TS): `index.html` + `popup.ts` |
| `src/lib/` | Общий код: `messages.ts` (контракт сообщений), `settings.ts` (BYOK) |

Docker Compose в проекте **нет** и не нужен (расширение полностью клиентское —
обоснование в `docs/TECHNICAL_PLAN.md §9`).

## Модель-политика (какую модель на что вызывать)

Зафиксировано пользователем. Придерживайся строго.

- **Sonnet 5** — анализ документации и основной флоу чата (диалог с
  пользователем по умолчанию, эта модель ведёт сессию); просмотр кода,
  тестирование.
- **Opus 4.8** — написание и планирование, глубокие рассуждения, независимое
  ревью. Через **Agent-тул с `model: "opus"`**.
- **Fable 5** — написание кода. Через **Agent-тул с `model: "fable"`**.

## Git / dev-workflow (обязательный процесс на каждый этап/задачу)

Главная ветка — **`master`** (не `main`).

1. **Новая ветка** от `master` на этап/задачу.
2. **Анализ** (чтение `docs/`, `SKILL.md`, текущего кода) — основной чат
   (**Sonnet 5**).
3. **Написание кода** по плану — Agent с **`model: "fable"`**.
4. **Просмотр** написанного кода — основной чат (**Sonnet 5**).
5. **Независимое ревью** — Agent с **`model: "opus"`** (независимый от агента
   из шага 3).
6. **Цикл исправлений:** если есть замечания — заново с шага 3, **не более
   3 циклов** шаги 3→5.
7. **Тестирование** — основной чат (**Sonnet 5**): `npm run build`
   (tsc + vite), `npm test` (после появления тестов).
8. **Финальный просмотр + актуализация документации** в `docs/` —
   **Sonnet 5**.
9. **По завершении этапа:** commit, push ветки, создание **PR в `master`**.

Коммить/пуш только когда этап завершён (или пользователь явно попросил).

## Команды разработки

```bash
npm install
npm run build        # tsc --noEmit + vite build → dist/ (должна быть зелёной с Этапа 0)
npm run dev          # dev-сборка с HMR (@crxjs)
npm run typecheck    # только проверка типов (tsc --noEmit)
npm test             # Vitest (после появления тестов, Этап 1+)

# Загрузка в браузер: chrome://extensions → Developer mode →
# Load unpacked → выбрать dist/
```

## Конвенции проекта

- **Язык:** документация и субъекты коммитов — **по-русски**; код,
  идентификаторы и комментарии в коде — по-английски.
- **Коммиты:** conventional-commit с русскими субъектами, напр.
  `feat(stage1): саммаризация всей страницы через LLM`. Завершать трейлером
  `Co-Authored-By: Claude`.
- **Ветки:** feature-ветки на этап; PR в `master`.
- **BYOK/секреты:** ключ пользователя — только в `chrome.storage` (runtime),
  **никогда** не хардкодить, не логировать, не коммитить. `dist/`,
  `node_modules/`, `.env*` — в `.gitignore`.
- Перед коммитом прогонять `npm run build` (и `npm test`, когда есть тесты).
- Не раздувать скоуп — см. `docs/POST_MVP_PLAN.md` (Excluded — Never).
